## Process failures

### (a) Worktree drift — ~90 min lost

Claude Code silently created `.claude/worktrees/determined-buck` instead of
editing the main working tree. All edits landed in the worktree's copy of
`webapp/index.html`, not the file served by `python -m http.server`. The live
preview never changed. Later, a second worktree `heuristic-borg` appeared.

Diagnosis took ~90 minutes of "why isn't my edit showing up" before realizing
the served file was untouched. Fix: added a sticky guardrail to every Claude
Code prompt: "Work directly on the main working tree. Do NOT create a git
worktree under `.claude/worktrees/`."

This is the same class of bug as the `relaxed-villani` worktree drift on
April 8. Must be enforced at prompt level — Claude Code does not respect
prior-session instructions about worktrees.

### (b) .env.save close-call

During morning setup, a `.env.save` file containing Sentinel Hub OAuth
credentials appeared in the staging area. Caught before commit. Root cause:
nano auto-backup. Added `.env.save` to `.gitignore`.

### (c) Git identity on Galaxy-A14

`git commit` failed on the Galaxy-A14 (Termux) because `user.name` and
`user.email` were not configured. Quick fix: `git config --local` in the repo.
Not a recurring issue but worth noting for multi-device workflows.

## What we learned

1. **Worktree drift is the #1 Claude Code footgun.** Two sessions in a row now.
   Must be guarded at prompt level, not just in CLAUDE.md.
2. **Custom Leaflet panes are essential for multi-overlay apps.** Without explicit
   z-index control, image overlays fight with GeoJSON layers and tile layers.
3. **Credential files multiply.** `.env`, `.env.save`, `.env.bak` — gitignore
   patterns need to be broad (`*.env*` or explicit list).
