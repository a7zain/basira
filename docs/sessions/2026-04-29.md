### Pass 8 — SQ1D Part B: Lolli formula spec + faithful compute

Read `research/dust-honesty/references/lolli_2024.pdf` (gitignored, retrieved 2026-04-28). Wrote `research/dust-honesty/docs/sq1d_lolli_formula.md` documenting the equation, band selection, denominator, masking rules, and every implementation choice that the paper underspecifies. Committed as a separate review-able artifact:

- `25a4233` — SQ1D Part B: Lolli 2024 faithful formula spec

Formula (Lolli 2024 Eq. 1, p. 13):

```
DBB(i,j) = (1/5) Σ_k [ρ_k^TOA(test, L1C) − ρ̄_k^TOA(ref, L1C)] / ρ̄_k(ref, L2A)
                                                            k ∈ {2,3,4,11,12}
```

Implementation choices flagged in the spec (none had a paper-defined answer):

1. **Test scene selection per (YYYY-MM, sub_aoi).** Paper uses fixed dates; we have monthly slots → picked lowest-CLOUDY_PIXEL_PERCENTAGE scene in calendar month over AOI bbox. `system:index` recorded in CSV `notes` column.
2. **Reference selection criterion.** Paper requires AERONET-confirmed AOD < 0.07; no AERONET in Riyadh, so substituted our locked UVAI-based references from `sq1d_references.json` (PRIMARY arm only — sensitivity alternates queued for Part B').
3. **Resolution.** Computed at 20 m natively (SWIR native; RGB sampled). Paper allows 10 m or 20 m; scene-mean is near-invariant.
4. **Cloud / invalid-pixel mask.** Paper gives no formal mask. Used SCL band, valid = {4, 5, 6, 7, 11}. Aggregation domain = `valid_test ∩ valid_ref ∩ not_water` where `not_water = ρ̄_12(L2A) ≥ 0.01`.
5. **L1C↔L2A pairing.** Match by `system:index`; fallback to lowest-cloud L2A on the same date if exact match missing. Skip-and-log if no L2A.
6. **Reflectance scale.** S2_HARMONIZED / S2_SR_HARMONIZED harmonize the post-2022 +1000 offset, so divide by 10000 only.
7. **Denominator no-data.** Pixels where any of the 5 L2A reference bands ≤ 0 are excluded (avoids divide-by-zero).

Implementation: `research/dust-honesty/scripts/sq1d_lolli_faithful.py`. Output: `research/dust-honesty/data/sq1d_dbb_faithful.csv` (30 rows; columns `date, sub_aoi, final_label, ref_date, dbb_faithful, n_valid_pixels, n_total_pixels, notes`).

Self-reference unit test: row `2020-04 / diriyah_gate` has `test_id == ref_id == 20200425T072609_..._T38RPN` and `dbb_faithful = 0` exactly. Confirms numerator-zero behaviour.

KSP monotone: clean median −0.022 → light_haze +0.064 → heavy_dust +0.218. Sign convention matches Lolli (dust → DBB > 0). Class separation real, magnitudes spread cleanly.

Qiddiya non-monotone: clean median +0.109 sits above light_haze median +0.100 (n=2 light_haze + reference-independent label contamination — see §Findings).

Old-vs-new median deltas per AOI: Diriyah +0.226, KSP +0.214, Qiddiya +0.354. Non-cosmetic shift in the right direction — old uniform negative around −0.22 to −0.32 had no atmospheric meaning; new straddles zero and orders by atmospheric severity.

Calibration-set expansion via UVAI-anchored positives (target n_pos ≥ 10 per AOI) is the right move. ## Lessons reinforced

From Part 1:

- **Stale intro templates miss pivots.** The intro pasted at session start was pre-2026-04-20-pivot framing. Reading CLAUDE.md and the master plan first surfaced the gap before any work happened. Worth normalizing into the workflow that the chat starts with the load and not the intro.
- **Methodology holes show up at the seam between pipeline steps.** UVAI in test caption was technically correct for review purposes but methodologically wrong for AI pre-label input. Caught at the right moment (before pre-labeling started). Cost of catching late would have been re-running the entire pre-label pass with disclosed methodology drift.
- **Cheap distribution-checks before expensive GEE renders.** Claude Code's instinct to run the count-verification step first saved an entire GEE budget cycle in case the source data was wrong.
- **Asymmetric calibration sets are silent failures.** If we'd relabeled only KSP, SQ1B would have run on 11 clean-relabeled + 13 contaminated + 6 OK rows. The threshold tuning would have been confounded by labels that were systematically wrong on Qiddiya. Catching this required pattern-matching across AOIs ("if construction-substrate confound applies at KSP it applies at Qiddiya too").
- **Force-add precedents earn their keep.** The `442d7b0` precedent for forcing past `research/dust-honesty/data/` gitignore meant every step today could write calibration data without arguing with `.gitignore`.
- **Visual review is part of verification.** The 2025-11-27 candidate render silent failure didn't trigger the stop-on-fail guard. Catching it required eyeballing the montage Ahmed uploaded to chat. "Verification discipline" includes looking at the images, not just trusting the success message.

From Part 2:

- **Self-reference unit test is the cheapest formula validator.** Diriyah row with `test_id == ref_id` returning `dbb_faithful = 0` exactly (not "≈") rules out numerical artifacts. Costs nothing to design in; pays back the moment the formula is implemented.
- **Wrong premises cost more than missing checks.** I (chat) wrote the Part B' self-reference check assuming the alternate ref dates were in the calibration set. They weren't (calibration set is a stratified sample; no requirement they would be). CC reported and continued correctly rather than stopping on a non-issue. Better to be precise about what's in the calibration set before writing checks against it.
- **AUC-vs-CI separation is the right way to read a STOP rule.** Original SQ1B stopped on both (AUC 0.55 + CI 0.026). Faithful re-run separates the two: AUC clears (0.82–0.84), CI doesn't (0.046–0.060). Different blockers point to different next moves — methodology was the original problem, sample size is now.
- **Don't over-extrapolate from a single-pass result on n=2.** Qiddiya's light_haze < clean inversion would have looked like a formula bug if I'd seen it once. Reading the same shape in the alternate (different reference, same 13 scenes) made it diagnostic — reference-independent → label contamination, not formula. The sensitivity check earned its keep here.
- **CDSE 503 is a multi-day pattern, not a transient.** Three instances this week (2026-04-28, 2026-04-28 retry, 2026-04-29). When pre-flight returns 503, abort and continue; do not retry-loop. CDSE OData fallback (raw netCDF, bypasses Sentinel Hub) is the eventual production path anyway per 2026-04-28 spike notes.
- **Bestbeffort + multiple reduceRegion calls = silent count drift.** The `n_valid > n_total` symptom in the original Part B output was bestEffort scale drift between two separate reducers. Single-image, single-reducer (sum + count), no bestEffort, identical scale is the safe pattern. Worth a guardrail mention.
- **Skip-with-marker beats raise-loudly for batch operations.** Original guard either raised (killing batch) or didn't fire (silent black thumbnail). New skip-with-marker: visibly unmissable failure indicator + batch survives. Standard going forward.
