### SQ1 — done, with thumbnail iterations
Ported Lolli's DBB index, pulled 30-scene sample stratified by season across
the three sub-AOIs. DBB values came out tight and uniformly negative
(−0.315 to −0.219, std 0.022) — expected over bright desert where SWIR >
visible year-round. Dust signal is the relative shift toward less-negative,
not sign change.

Thumbnail render needed two iterations before it was usable for labeling:

1. **First render had black polygon-mask frames.** Phase 1 source tifs are
   polygon-clipped at download time — using raw raster bbox would have just
   shown the polygon hole. Claude Code instead computed the largest
   axis-aligned all-valid interior rectangle per scene. Clean rectangular
   tiles, 100% real pixels, but only 9-34% of each raster's footprint.
   Acceptable for atmospheric labeling — we're judging clarity, not
   boundaries. Commit `5bdb106`.

2. **Second render used a global stretch.** The three sub-AOIs have very
   different baseline albedos (qiddiya = pale desert, diriyah = dense
   urban). A global 2/98 percentile compressed within-AOI atmospheric
   variation into a narrow band. Fixed by computing the stretch separately
   per AOI, applied only within that AOI's tiles. Commit `91a7077`.

Both fixes were Claude-pushed; I would have ground through with the
black-frame and global-stretch versions without realizing they were hurting
label quality.

### SQ1 manual labels — AI-pre-label, researcher-review
This is the workflow change worth remembering. I bounced off the cold
labeling task. Claude refused to just label everything for me (rightly:
threshold calibrated against AI judgment is a credibility hole in a
research piece whose pitch is "Sen2Cor isn't honest"). Compromise: Claude
pre-labels the 30 scenes against a written rubric, with confidence ratings
and reasoning per tile; I review the call sheet in 5 minutes against the
full-resolution PNG. I'm the labeler of record; Claude is a first-pass
assistant.

The first pre-labeling pass was done from a downsampled screenshot — Claude
flagged 8 low-confidence calls. I then uploaded the full PNG. Higher
resolution let Claude re-judge those 8, plus catch two high-confidence
calls that turned out to be wrong against the cleaner image (notably
2024-08 KSP — what looked like dust was actually bare construction
substrate at megaproject sites).

Final distribution: 23 clean / 4 light_haze / 2 heavy_dust / 1 cloud /
0 mixed.

Methodology footnote that goes in every B writeup:
"The 30-scene calibration set was pre-labeled by an AI assistant (Claude)
against a written rubric for atmospheric clarity, then reviewed and
finalized by the researcher with focus on low-confidence calls."

Labels CSV force-added past `.gitignore` — calibration data must be

### SQ1B — stop rule fired
Bootstrap 95% CI half-width came in at 0.026 vs the 0.020 threshold.
AUC 0.55 (conservative mapping) and 0.51 (aggressive). The conservative
threshold sat at the very top of the DBB distribution — moving any one
labeled scene by one rank would flip it. Genuinely unstable.

Diagnosis: class imbalance. Only 2 heavy_dust positives in 30 scenes is
below what any binary classifier can stabilize on. Not a model failure —
a calibration-set failure. Lolli formulation also flagged as unverified
(MDPI and ResearchGate both 403'd to automated fetch this session;
direction-of-signal sanity check passed in SQ1 so threshold tuning is
still well-posed regardless of formulation).

No threshold shipped. `dbb_index.py` unchanged. `is_dust_flagged` not
written. Commit `3d3b511`. The stop rule worked as designed.

### SQ1C — launched, blocked
Path forward: expand calibration set with UVAI-anchored positives.

Strategy:
1. Pull TROPOMI monthly UVAI (mean + max) for Riyadh AOI 2020-2026 — uses
   existing `fetch_tropomi.py` pattern via Sentinel Hub Statistical API
2. Rank months by max UVAI, identify top 15 not already in the SQ1 sample
3. Pull Sentinel-2 monthly mosaics for those 15 months × 3 sub-AOIs (up to
   45 candidate scenes, expect ~30-40 with valid tifs)
4. Compute DBB for each, generate per-AOI-stretched thumbnail grid with
   UVAI evidence visible in captions
5. Stop. I label the new scenes via the same AI-pre-label + review workflow.
6. Re-run SQ1B threshold sweep on combined ~60-70 scene set.

Critical design choice: UVAI selects WHICH months to pull, but does NOT
label the scenes. Visual labels remain the calibration target. If UVAI
were the label, SQ3 (which validates the threshold against UVAI) would
become circular.

Stop-rule clause built in: if fewer than 8 new scenes get produced, SQ1C
halts before thumbnail generation and reports diagnostic info.
