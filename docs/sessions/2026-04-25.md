## Bugs caught (in order)

- Per-channel P2–P98 RGB stretch produced over-saturated "tourism
  brochure" output. Fixed with joint P1–P99 + gamma 1.4. Sentinel-2
  reflectance is linear; mild gamma is what every serious EO RGB
  pipeline does.
- Default 4 fps timelapse felt slow on screen. Bumped to 6 fps.
- CSS custom-property URL resolution: `--aoi-mask: url(...)` set
  inline resolves relative to the *consuming* stylesheet, not the
  document. Fixed by setting `mask-image` directly inline so URLs
  resolve from the document root.
- Chromium defaults grayscale PNG masks to alpha-mode, not luminance.
  Required explicit `mask-mode: luminance` + `-webkit-mask-mode`.
- Backdrop generated from polygon-clipped GeoTIFFs was black outside
  the polygon. Rewrote `phase1_extract_backdrops.py` to use existing
  wide-context jpgs from `assets/backdrops/<aoi>_context.jpg`.
- IntersectionObserver `threshold: 0.2` unreachable on tall sections
  in small viewports (max ratio = viewport/section-height). Switched
  to `threshold: 0` + `rootMargin: "0px 0px -15% 0px"`.
- Stale diagnostic print statement in `build_frames` (per-band
  `enumerate(fixed_lo_hi)`) blocked the joint-stretch refactor and
  made the first batch of MP4 renders fail silently. Caught before
  shipping stale 4 fps MP4s.

## Lessons (capture for memory)

- Spec contradictions inside a single locked v1 are real. The
  §Imagery vs §Chapter 2 wording landed yesterday; today's session
  caught it because Claude Code surfaced the conflict before
  building, not after. Verification discipline working.
- **Trust drift.** At one point Claude (this chat) and I both
  treated "commit clean + push clean" as equivalent to "deployed
  site live." It wasn't — Pages was caching aggressively and showed
  stale content for ~30 minutes after the fix. For Phase 1
  deliverables, the public URL *is* the artifact. Verifying commits
  ≠ verifying deployment. Always open `a7zain.github.io/basira/`
  myself, end of every session, before closing.
- **Audit drift.** The April 20 cleanup audit said root `index.html`
  was a 204b redirect. Today it was a 38 KB Phase 4 landing page.
  Audits go stale fast; trust `ls -la` over memory.
- **Misdiagnosis caught by ground-truth.** Today's Pages-source
  confusion (Claude Code reported `/webapp`, was actually `/root`)
  was caught only because I said "Pages are already on main-/root."
  Trust ground-truth checks over agentic summaries.
