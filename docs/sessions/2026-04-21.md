## Key decisions (do not re-litigate)

- **Cinematic NYT-style direction, not dashboard-style.** Committed after explicit
  pushback discussion. Every future scoping decision passes through: "does this
  belong in a New York Times feature?" If no, it doesn't go on the main scroll.
  Phase 4 analytics, metrics grids, filter panels, toggles — none of these go
  on the main surface. They can live in the methodology chapter, the memo, or
  the repo.
- **Single-page scroll over separate pages.** Simpler to build, more cinematic
  when done right, better on mobile, no page-transition clunk.
- **KML polygons over bounding boxes.** Sentinel Hub Process API accepts
  GeoJSON geometry directly; downloads are polygon-clipped. Reduces PU slightly,
  produces cleaner outputs.
- **Arabic بصيرة retained as name mark only.** No bilingual body copy. Authentic
  naming without QA burden.
- **Repo rename pending: sar-change-detection → basira.** URL will become
  `a7zain.github.io/basira`. Not done this session; do at site-build time.
- **KSP kept at full polygon, not tightened to Royal Arts Complex.** Claude
  walked back the earlier "tighten it" suggestion after seeing the Google Earth
  polygon — the greening story is the full park, not a sub-zone.

## Observations and learnings

### Geocoding from memory is unreliable

Claude's initial bbox guess for Qiddiya was ~8-10 km off from the actual
entertainment district. Lesson: for any AOI-defined work, human-drawn polygons
are the right starting point, not coordinate estimates. The KML pipeline is
the pattern to repeat.

### Concurrent Sentinel Hub requests are tolerated

Chain-script edge case: `kill -0 $QPID` picked up a transient shell PID that
exited before Python started, so KSP began in parallel with qiddiya_core for
~40 minutes. Sentinel Hub handled it without rate-limiting or 429s. Future
runs: either `wait` on the known Python PID, or just accept concurrency since
SH tolerates it. Not a blocker.

### Cinematic is a binding constraint, not an aesthetic

Committing to NYT-feature direction made the site spec stricter, not looser.
The surface-discipline list in `site_spec.md` is explicit about what stays off
the main scroll; that discipline only exists because the direction made the
no's easier to say.

### Writing-with-Claude requires raw material from Ahmed

For the cinematic prose to land, the voice must be Ahmed's, not Claude's. The
working mode is: Ahmed drafts messy first pass (voice notes, typos, mixed
EN/AR fine); Claude cleans and tightens; Claude never invents voice. This is
flagged ahead of Chapter 5 (methodology) because that's where it matters most.
